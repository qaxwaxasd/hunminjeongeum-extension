# UI 설계 명세서: 초록색 포인트 테마 (다크 / 라이트 모드)

본 문서는 서비스의 일관된 사용자 경험을 위해 정의된 **디자인 시스템**과 **모드별 환경 설정**, 그리고 **기능별 UI 적용 가이드**를 포함하고 있습니다.

## 1. 디자인 시스템: 초록색 포인트 (Point Color)
포인트 컬러는 너무 진하지 않으면서도 다크 모드와 라이트 모드 양쪽에서 가독성을 확보할 수 있는 값을 사용합니다.

| 항목 | 다크 모드 (Dark) | 라이트 모드 (Light) | 설명 (Hex/예시) |
| :--- | :--- | :--- | :--- |
| **Primary (Point)** | `#52C41A` | `#389E0D` | 부드럽고 생동감 있는 초록 |
| **Primary-Hover** | `#73D13D` | `#237804` | 버튼 위 마우스 호버 시 |
| **Sub-Highlight** | `#B7EB8F` (20% 투명도) | `#F6FFED` | 텍스트 배경 강조 시 |

## 2. 모드별 배경 및 텍스트 설정
사용자의 시스템 설정에 따라 자연스럽게 어울리는 배경색과 텍스트 색상 구성입니다.

| 요소 | 다크 모드 (Dark) | 라이트 모드 (Light) | 비고 |
| :--- | :--- | :--- | :--- |
| **Base BG** | `#111827` (Gray-900) | `#F9FAFB` (Gray-50) | 전체 배경 |
| **Surface (Card)** | `#1F2937` (Gray-800) | `#FFFFFF` (White) | 교정 팝업 카드 |
| **Main Text** | `#FFFFFF` | `#111827` (Gray-900) | 기본 글자색 |
| **Sub Text** | `#9CA3AF` (Gray-400) | `#6B7280` (Gray-500) | 설명 및 메타 정보 |
| **Error Line** | `#FF4D4F` | `#CF1322` | 오타 밑줄 (초록과 대비) |

## 3. 기능 요구사항 (FR)에 따른 UI 적용 가이드
주요 기능별로 초록색 포인트(Green Point)를 시각적으로 적용하는 방법입니다.

| 번호 | 기능 핵심 | UI 설계 가이드 (Green Point) |
| :--- | :--- | :--- |
| **FR-01~03** | 실시간 검사 및 밑줄 | 오타(빨간 밑줄)와 대비되도록 문법 제안은 초록색 또는 파란색 점선으로 표시함. |
| **FR-04~05** | 교정 카드 및 수정 | 수정 버튼은 초록색 배경에 흰색 글씨를 사용하여 '완료'와 '긍정'의 의미 전달. |
| **FR-06** | 나의 사전 등록 | 사전 추가 버튼 클릭 시 아이콘이 초록색으로 활성화되며 피드백 제공. |
| **FR-07** | 서비스 ON/OFF | 토글 스위치가 'ON' 상태일 때 부드러운 초록색으로 점등됨. |
| **FR-10** | 오류 통계 확인 | 통계 바 차트에서 '정상 문장' 혹은 '교정 완료' 비중을 초록색 그래프로 표현. |
